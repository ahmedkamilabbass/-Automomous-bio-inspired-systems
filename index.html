Setup Guide: Your Robotics Development Environment

This comprehensive guide will walk you through setting up your computer for this course. We'll install a virtual machine, Ubuntu Linux, ROS 2 Jazzy, the Webots simulator, and the necessary bridge to make them work together. A stable and consistent environment is key to your success!

1. Virtual Machine Software (VirtualBox)

Using a Virtual Machine (VM) keeps your robotics setup isolated from your main operating system, provides easy recovery with snapshots, and ensures everyone in the class has a consistent environment.

Prerequisites:

  • Host computer (Windows, macOS, Linux) with at least 8GB RAM (16GB recommended).
  • ~50-100GB free disk space.
  • Internet connection.
  • **Crucially:** Enable Virtualization Technology (VT-x/AMD-V) in your computer's BIOS/UEFI settings.

Steps:

  1. Download VirtualBox from virtualbox.org/wiki/Downloads. Choose the appropriate installer for your **host operating system**.
  2. Run the downloaded installer and follow the on-screen prompts. Generally, accept the default settings.
  3. Allow any network adapters or drivers to be installed.
  4. **Important:** Reboot your host computer after installation if prompted.

2. Creating Your Ubuntu 24.04 Virtual Machine

Ubuntu Linux is the standard operating system for ROS development. We will set up a new virtual machine specifically for our robotics work.

Steps:

  1. Download the `Ubuntu 24.04 LTS (Noble Numbat)` desktop ISO image (approx. 6.4 GB) from ubuntu.com/download/desktop.
  2. Open VirtualBox Manager. Click **"New"** (or Machine > New).
  3. Set **Name:** `Ubuntu-24.04-Robotics` (or similar).
  4. Select **ISO Image:** Choose the downloaded Ubuntu 24.04 LTS ISO.
  5. Set **Type:** `Linux`, **Version:** `Ubuntu (64-bit)`.
  6. **Base Memory (RAM):** **Minimum 4096 MB (4GB)**. (8192 MB / 8GB recommended if host allows). Stay within the green zone.
  7. **Processors (CPU):** **Minimum 2 CPUs**. (4+ recommended).
  8. **Hard Disk:** "Create a virtual hard disk now". Select `VDI (VirtualBox Disk Image)` and `Dynamically allocated`. Set **Size:** **Minimum 50 GB**. (80-100GB recommended).
  9. Click **"Finish"**.

3. Installing Ubuntu Desktop 24.04 LTS

Now we will install the Ubuntu operating system onto your newly created virtual machine.

Steps:

  1. In VirtualBox, select your new VM and click **"Start"**.
  2. Follow the Ubuntu Installer prompts:
    • Choose your language.
    • Select **"Install Ubuntu"**.
    • Select your Keyboard Layout.
    • Connect to the internet (important for updates).
    • **Installation Type:** Select **"Normal installation"**, **"Download updates while installing Ubuntu"**, and **"Install third-party software"**.
    • **Erase disk and install Ubuntu:** This is safe within a VM.
    • Select your Location/timezone.
    • Create your User Details (username, computer name, password). **Remember this password!**
  3. Wait for the installation to complete. Reboot when prompted.
  4. **Install VirtualBox Guest Additions:** After Ubuntu installs, in the VM window, go to `Devices > Insert Guest Additions CD Image...`. Follow the prompts inside the VM to install for better display/mouse integration. Reboot VM again.

4. ROS 2 Jazzy Jalisco Installation

ROS (Robot Operating System) is a flexible framework for writing robot software. We'll install ROS 2 Jazzy Jalisco.

Steps:

  1. **Open a Terminal** in your Ubuntu VM (`Ctrl+Alt+T`).
  2. **Set Locale (if needed):**
    locale # check for UTF-8 support
    sudo apt update && sudo apt install locales
    sudo locale-gen en_US en_US.UTF-8
    sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
    export LANG=en_US.UTF-8
  3. **Enable ROS 2 repositories:**
    sudo apt install software-properties-common
    sudo add-apt-repository universe
    sudo apt update && sudo apt install curl -y
    export ROS_APT_SOURCE_VERSION=$(curl -s https://api.github.com/repos/ros-infrastructure/ros-apt-source/releases/latest | grep -F "tag_name" | awk -F\\" '{print $4}')
    curl -L -o /tmp/ros2-apt-source.deb "https://github.com/ros-infrastructure/ros-apt-source/releases/download/${ROS_APT_SOURCE_VERSION}/ros2-apt-source_$(. /etc/os-release && echo $VERSION_CODENAME)_all.deb"
    sudo apt install /tmp/ros2-apt-source.deb
  4. **Install ROS 2 Jazzy Desktop:**
    sudo apt update
    sudo apt upgrade
    sudo apt install ros-jazzy-desktop
  5. **Setup Environment:**
    echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
    source ~/.bashrc
  6. **Test ROS 2:**
    • Open Terminal 1: `ros2 run demo_nodes_cpp talker`
    • Open Terminal 2: `ros2 run demo_nodes_py listener`
    • You should see messages being sent and received.

5. Webots R2025a Installation

Webots is our 3D robot simulator. We will install the latest stable version, R2025a.

Steps:

  1. **Open a Terminal** in your Ubuntu VM.
  2. **Method 1: APT Repository (Recommended for updates)**
    1. **Add Cyberbotics GPG Key:**
      sudo mkdir -p /etc/apt/keyrings
      cd /etc/apt/keyrings
      sudo wget -q https://cyberbotics.com/Cyberbotics.asc
    2. **Add Webots Repository:**
      echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/Cyberbotics.asc] https://cyberbotics.com/debian binary-amd64/" | sudo tee /etc/apt/sources.list.d/Cyberbotics.list
      sudo apt update
    3. **Install Webots:**
      sudo apt install webots
  3. **Method 2: Direct Debian Package (.deb)**
    1. **Download .deb file:** Go to cyberbotics.com/#download and download the `Webots R2025a_amd64.deb` package for Ubuntu.
    2. **Install:**
      cd ~/Downloads
      sudo apt install ./webots_R2025a_amd64.deb
  4. **Test Webots:** Open Webots from your applications menu or by typing `webots` in the terminal.

6. Connecting Webots & ROS 2: `webots_ros2`

This package acts as the bridge, allowing your ROS 2 nodes to communicate seamlessly with Webots simulations.

Steps:

  1. **Open a Terminal** in your Ubuntu VM.
  2. **Installation (from binaries - Recommended):**
    sudo apt update
    sudo apt install ros-jazzy-webots-ros2
  3. **Set `WEBOTS_HOME` (Important for ROS 2):**
    • The default Webots installation via APT is `/usr/local/webots`.
    • Add this to your `~/.bashrc` to ensure ROS 2 can find Webots:
      echo "export WEBOTS_HOME=/usr/local/webots" >> ~/.bashrc
      source ~/.bashrc

7. Final Test: Running a Webots-ROS 2 Example

This test verifies that your Webots and ROS 2 installations are successfully integrated.

Steps:

  1. **Open a new Terminal** in your Ubuntu VM.
  2. **Source ROS 2 environment:**
    source /opt/ros/jazzy/setup.bash
  3. **Run a `webots_ros2` demo launch file:**
    ros2 launch webots_ros2_universal_robot universal_robot_launch.py
  4. **Observe:** A Webots window should open with a Universal Robot, and ROS 2 nodes should be running in the terminal. This confirms a successful setup!

Common Issues & Troubleshooting

Don't worry if you hit a snag! Here are common problems and their solutions:

  • **Virtualization Not Enabled:** VM won't boot 64-bit OS.
    • **Fix:** Enable VT-x/AMD-V in your computer's BIOS/UEFI settings.
  • **Incompatible Python/ROS/Webots Versions:** Errors during install or runtime.
    • **Fix:** Ensure you're strictly following the guide for Ubuntu 24.04, ROS Jazzy, and Webots R2025a, as these versions are compatible.
  • **Missing `WEBOTS_HOME`:** `webots_ros2` cannot find Webots.
    • **Fix:** Set `export WEBOTS_HOME=/usr/local/webots` in `~/.bashrc` and `source ~/.bashrc`.
  • **Missing Dependencies:** Errors during `apt install` or `colcon build` (if building from source).
    • **Fix:** Read error messages carefully; they usually tell you the missing package name. Use `sudo apt install `.
  • **Graphics/Display Issues in VM:** Poor performance, no full screen.
    • **Fix:** Ensure VirtualBox Guest Additions are installed. In VM settings (Display), enable 2D/3D acceleration and allocate more video memory.
  • **Internet/Network Issues in VM:** Cannot download packages.
    • **Fix:** Check VirtualBox network adapter settings for the VM; NAT is usually sufficient.

Why Webots?

This section introduces Webots as the foundational tool for our practical course. Its professional-grade features, open-source accessibility, and powerful integrations make it an ideal platform for bridging theory with hands-on application in robotics research and development.

💻

Open-Source & Accessible

Free for all students on any platform (Windows, macOS, Linux). This removes barriers and allows you to install and experiment on your own machine, a core part of our hands-on approach.

⚙️

Versatile & Powerful

Features a rich library of robots, sensors, and objects. You can program in Python, C++, Java, and more, ensuring the skills you learn are valuable and transferable to the wider robotics field.

🔗

ROS & Industry Ready

Integrates seamlessly with the Robot Operating System (ROS), the industry standard for robotics development. This prepares you for advanced topics, including multi-robot systems and swarm intelligence.

Your Learning Path

This interactive learning path outlines the two foundational lectures. Click on each step to expand its content and explore the core concepts you will master. The journey is designed to build your skills progressively, from basic navigation to programming intelligent robot behaviors.

1
System Readiness Check

Before we begin, we ensure your system is ready. The first lectures cover installing Ubuntu, Webots, and ROS. A key check is Python compatibility; for instance, Webots R2022b works with Python 3.7-3.10. Mismatched versions are a common source of errors.

Creating an Environment

You'll learn to craft simulation worlds from scratch. This involves adding fundamental nodes like `Background` and `RectangleArena`, and importing pre-built robots like the e-puck. You will learn to manipulate objects directly in the 3D view and make structural changes through the Scene Tree.

The DEF-USE Mechanism

A key concept for efficiency is the `DEF-USE` mechanism. It allows you to define an object once (`DEF`) and reuse it multiple times (`USE`). Changes to the `DEF` node automatically update all instances, which is essential for creating complex scenes and later, for deploying swarms of identical robots.

2
Robot Controllers

A controller is the brain of your robot. We'll focus on Python. The core of any controller is the main loop containing `robot.step(TIME_STEP)`. This function is critical: it synchronizes your code with the simulator's physics engine, reads new sensor data, and applies your motor commands.

Sensors and Actuators

Robots perceive and act through devices. You'll learn to access sensors (like `DistanceSensor` or `Camera`) and actuators (like `Motor` or `LED`) using `robot.getDevice("name")`. Sensors must be enabled with `sensor.enable(sampling_period)` to get data. Motor commands like `motor.setVelocity()` are only executed at the next `robot.step()` call.

Implementing Behaviors

From simple movements like driving forward, you will progress to creating reactive behaviors. A classic first step is obstacle avoidance, where the robot uses its distance sensors to detect objects and changes its motor speeds to navigate around them. This introduces the fundamental concept of a sense-plan-act cycle.

Groundwork for Swarm Robotics

These foundational lectures prepare you for one of the most exciting areas of bio-inspired robotics: swarm intelligence. Here are the key Webots features you'll use to design, simulate, and analyze the behavior of multi-robot systems.

📡

Inter-Robot Communication

Robots in a swarm communicate using `Emitter` and `Receiver` nodes. You can simulate different communication types, like line-of-sight "infra-red" or broadcast "radio," to test the robustness of your algorithms.

👑

Supervisor Controllers

A special `Supervisor` controller can oversee the entire simulation. It can reset the world, move robots, and send data to the swarm, making it essential for running automated experiments like genetic algorithms.

⚡️

Performance Optimization

Simulating large swarms is computationally intensive. You'll learn to balance realism and speed by simplifying collision models (`boundingObject`) and adjusting the simulation time step (`basicTimeStep`).
